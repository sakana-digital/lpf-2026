# 守ること

- 日本語で話すこと
- 文章を views にハードコードせず，vue-i18n で別ファイルに書く
- views を勝手に増やさない
- CSS nesting を積極的に使う
- コンポーネントと直接関係のない部分は別ファイルに分ける
- コメントはユーザーのコンテキストを直接書かない
- CSS, HTML の部分には基本的にコメントは追加せず，複雑な JS などにはつけてもよい
- 開発サーバーはユーザーが localhost:5173 ですでに起動している場合が多いため，確認時はそれを使う
- 検証は `vp`, `bun` を使うこと
- Commit メッセージは prefix をつけた上で英語
- Commit を戻すときは指示した ID がそのブランチの先頭になるようにすること
- すでに remote に push されてしまっている場合は実行しない
- Merge のメッセージは必ずデフォルト，--no-ff
